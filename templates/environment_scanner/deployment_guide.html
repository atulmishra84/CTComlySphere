Deployment Methods
Kubernetes Deployment

Deploy across K8s clusters for comprehensive container monitoring.

Prerequisites:
  • Kubernetes 1.19+ cluster
  • kubectl access with admin privileges
  • Network access to platform API
Installation:
kubectl apply -f https://platform.healthcare-ai.com/deploy/k8s/
kubectl create secret generic scanner-config \
--from-literal=platform-url="https://your-platform.com" \
--from-literal=api-key="your-api-key"
Docker Deployment

Single-host Docker deployment for development and small environments.

Prerequisites:
  • Docker Engine 20.10+
  • Docker Compose (optional)
  • Access to Docker daemon socket
Installation:
docker run -d \
--name healthcare-ai-scanner \
-v /var/run/docker.sock:/var/run/docker.sock \
-e PLATFORM_URL="https://your-platform.com" \
-e API_KEY="your-api-key" \
healthcare-ai/scanner:latest
Cloud Deployment

Deploy across AWS, Azure, and GCP for multi-cloud monitoring.

Supported Platforms:
AWS Azure GCP
Terraform Deployment:
terraform init
terraform plan -var="platform_url=https://your-platform.com"
terraform apply
Agent-Based Deployment

Install lightweight agents on individual servers and workstations.

Supported OS:
Linux Windows macOS
Installation:
curl -sSL https://platform.healthcare-ai.com/install.sh | bash
sudo systemctl enable healthcare-ai-agent
sudo systemctl start healthcare-ai-agent
Configuration & Security
Required Configuration
Variable Description
PLATFORM_URL Healthcare AI Platform endpoint
API_KEY Authentication API key
ORG_ID Your organization identifier
SCAN_INTERVAL Scan frequency (default: 15m)
Security Considerations
  • Network Security: All communication uses TLS 1.3
  • Authentication: API keys with role-based access
  • HIPAA Compliance: No PHI data transmitted
  • Data Privacy: Metadata-only scanning
Important: Store API keys securely using your organization's secret management system.
Monitoring & Troubleshooting
Health Monitoring

Monitor scanner agent health and performance:

# Check agent status
curl -H "Authorization: Bearer $API_KEY" \
https://your-platform.com/api/agents/status

# View agent logs
kubectl logs -l app=healthcare-ai-scanner
Common Issues
  • Verify PLATFORM_URL is correct
  • Check API_KEY validity
  • Ensure network connectivity to platform
  • Check firewall rules for outbound HTTPS
  • Verify scanner has access to target systems
  • Check service account permissions
  • Review scanner configuration filters
  • Ensure AI workloads have proper labels
